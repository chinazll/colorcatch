# 荣耀糖糖 — 技术设计文档

> **项目**: 荣耀糖糖 (Glory Candy)
> **基于**: ColorCatch 现有架构
> **版本**: v1.0

---

## 1. 渲染管线

### 绘制顺序（从后到前）

```
[1] 背景层  Background        — 静态深蓝渐变 + 星空粒子（只随时间动画，不随相机移动）
[2] 平台层  Platform          — 所有平台，在 world 坐标绘制
[3] 星星层  Star              — 金色收集物，在 world 坐标绘制
[4] 角色层  Player            — 可爱发光糖果战士，在 world 坐标绘制
[5] 粒子层  ParticleSystem    — 跳跃/收集/拖尾粒子，在 world 坐标绘制
[6] HUD层  HUD               — 屏幕空间（screen space），cameraY=0
```

**Camera Transform**: 所有 world 层对象在绘制时统一应用 `ctx.translate(0, -camY)`，HUD 不受影响。

**与 ColorCatch 的差异**:
- ColorCatch 的背景是动态云/山/糖果，荣耀糖糖改为深色星空主题
- 新增独立的星星层（ColorCatch 的 bonus 是附着在 platform 上的球）
- HUD 始终固定在屏幕顶部，不随相机滚动

## 2. 物理系统

### 2.1 核心概念

荣耀糖糖的核心物理模型是**弹性球**（bouncing ball），与 ColorCatch 的"重力+跳跃"模型完全不同。

### 2.2 物理参数

```javascript
// 物理常量
const GRAVITY        = 0.35;   // 向下的恒定加速度
const BOUNCE_DAMPING = 0.82;  // 碰平台后 vy 反向并乘以此系数（弹性损失）
const FRICTION       = 0.985;  // 水平速度每帧摩擦衰减
const MAX_FALL_SPEED = 18;     // 限制最大下落速度，防止穿模
const HORIZONTAL_IMPULSE = 4.5; // 点击左/右时的水平速度增量
```

### 2.3 弹跳物理逻辑（伪代码）

```javascript
// === Player 物理更新（每帧）===
function updatePhysics(dt) {
  // 1. 重力：持续把 vy 往下拉（正值=向下）
  this.vy += GRAVITY * 60 * dt;

  // 2. 限制最大下落速度
  if (this.vy > MAX_FALL_SPEED) this.vy = MAX_FALL_SPEED;

  // 3. 水平摩擦
  this.vx *= FRICTION;

  // 4. 速度积分
  this.x += this.vx;
  this.y += this.vy;
}

// === Platform 碰撞检测 ===
function checkPlatformCollision(player, platform) {
  // 碰撞条件：
  //   1. player 正往下掉（vy > 0）
  //   2. player 的下边缘上一帧在 platform 上方，本帧进入 platform
  //   3. player 水平方向和 platform 有重叠
  const prevBottom = player.y + player.radius - player.vy;
  const curBottom  = player.y + player.radius;

  if (player.vy > 0 &&                     // 正在下落
      prevBottom <= platform.y + 2 &&      // 上一帧在平台上（含2px容差）
      curBottom  >= platform.y &&           // 本帧已进入平台
      player.x + player.radius > platform.x &&
      player.x - player.radius < platform.x + platform.width) {

    // 吸附到平台表面
    player.y = platform.y - player.radius;

    // 反弹：vy 取反（向上弹）
    player.vy = -Math.abs(player.vy) * BOUNCE_DAMPING;

    return true; // 碰撞发生
  }
  return false;
}

// === 碰墙反弹 ===
if (player.x < player.radius) {
  player.x = player.radius;
  player.vx = Math.abs(player.vx) * 0.7; // 右侧弹回
}
if (player.x > W - player.radius) {
  player.x = W - player.radius;
  player.vx = -Math.abs(player.vx) * 0.7;
}
```

### 2.4 关键设计决策

**碰平台后 vy 直接取反**：荣耀糖糖的弹跳应该是"标准弹性球"行为——碰平台就弹起，方向变为向上（vy 变负），与 ColorCatch 的"落地才能再跳"完全不同。不需要维持"在空中才能跳"的状态。

**为什么不累积跳跃速度**：角色天然在弹跳，不需要用户按键触发跳跃。用户只能控制左右方向。

**vy 初始值**：游戏开始时给 `vy = -12`（初始向上弹），之后全靠平台碰撞反弹。

## 3. 触摸输入

### 3.1 输入设计

单手操作，极简：
- **点击屏幕左侧** → `vx -= HORIZONTAL_IMPULSE`（向左跳）
- **点击屏幕右侧** → `vx += HORIZONTAL_IMPULSE`（向右跳）
- 每次点击触发一次，不能连续叠加（冷却 100ms）

```javascript
// === 触摸处理 ===
_canvas.addEventListener('touchstart', (e) => {
  e.preventDefault();
  const rect = this.canvas.getBoundingClientRect();
  const touchX = e.touches[0].clientX - rect.left;
  const worldTouchX = touchX * (this.W / rect.width); // 映射到游戏坐标

  if (this.state !== STATE.PLAYING) return;

  const now = performance.now();
  if (now < this._inputCooldown) return; // 冷却保护

  if (worldTouchX < this.W / 2) {
    this.player.vx -= HORIZONTAL_IMPULSE;
    this._emitJumpParticles(this.player.x, this.player.y, 'left');
  } else {
    this.player.vx += HORIZONTAL_IMPULSE;
    this._emitJumpParticles(this.player.x, this.player.y, 'right');
  }

  this._inputCooldown = now + 100; // 100ms 冷却

  // 音效
  playBounce();
});
```

### 3.2 键盘支持（开发/PC）

```javascript
window.addEventListener('keydown', (e) => {
  if (this.state !== STATE.PLAYING) return;
  if (e.code === 'ArrowLeft'  || e.code === 'KeyA') this.player.vx -= HORIZONTAL_IMPULSE;
  if (e.code === 'ArrowRight' || e.code === 'KeyD') this.player.vx += HORIZONTAL_IMPULSE;
});
```

## 4. 摄像机系统

### 4.1 相机行为

```javascript
// 目标：角色永远保持在屏幕下方 40% 位置
const PLAYER_SCREEN_RATIO = 0.65; // 角色 y 占屏幕高度的 65%

_updateCamera() {
  // 角色在屏幕上的目标 y 坐标
  const targetScreenY = this.H * PLAYER_SCREEN_RATIO;

  // 相机的世界 y 坐标 = 角色世界 y - 目标屏幕 y
  let targetCamY = this.player.y - targetScreenY;

  // 【关键】相机只能向上移动（camY 变小），不能向下回退
  if (targetCamY < this.camY) {
    // 向上跟随：使用 lerp 平滑过渡
    this.camY = lerp(this.camY, targetCamY, CAM_LERP); // CAM_LERP = 0.08
  }
  // 如果 targetCamY >= this.camY，保持当前 camY（不向下退）
  // 这确保角色一旦上去，就不会因为自然下落而把相机往下带
}
```

### 4.2 游戏结束判断

```javascript
// 角色在世界坐标中掉到屏幕底部以下 GAME_OVER_THRESHOLD = 200 像素时，死亡
const playerScreenY = this.player.y - this.camY;
if (playerScreenY > this.H + 100) {
  this._triggerGameOver();
}
```

**注意**：由于相机只升不降，正常游戏过程中 `playerScreenY` 不会回到很高（角色一直在屏幕下半区）。角色只有在起始阶段向上弹跳不够高、且没有碰到平台时才会坠落出屏幕。

## 5. 平台生成

### 5.1 生成策略

```javascript
// 初始平台：在屏幕底部生成一排宽阔的平台确保玩家能开始
_generateInitialPlatforms() {
  // 第一个平台：居中，宽 160px（给玩家足够初始空间）
  this.platforms.push(new Platform(W/2 - 80, H - 60, 160, 'normal'));

  // 在此之上按间距生成
  let y = H - 60;
  for (let i = 0; i < 15; i++) {
    y -= randInt(GAP_MIN, GAP_MAX); // GAP_MIN=55, GAP_MAX=90
    const width = randInt(70, 120);
    const x = randInt(10, W - width - 10);
    this.platforms.push(new Platform(x, y, width, 'normal'));
  }
  this.nextPlatformY = y;
}

// 无限向上生成
_generateMorePlatforms() {
  const cfg = this._getCurrentDifficulty();
  while (this.nextPlatformY > this.highestY - this.H - 600) {
    this.nextPlatformY -= randInt(cfg.gapMin, cfg.gapMax);

    const width  = randInt(cfg.widthMin, cfg.widthMax);
    const x     = randInt(10, W - width - 10);
    const type  = this._randPlatformType(cfg);
    const color = this._randPlatformColor();

    const plat = new Platform(x, this.nextPlatformY, width, color, type);
    this.platforms.push(plat);

    // 在此平台上/旁生成星星（30% 概率）
    if (Math.random() < 0.30) {
      this._spawnStarNear(plat);
    }
  }
}

// 清理已离开视野的平台
_removeOffscreenPlatforms() {
  // 当平台上边缘 < camY - 200 时删除
  this.platforms = this.platforms.filter(p =>
    p.y < this.camY + this.H + 200
  );
}
```

## 6. 星星生成

### 6.1 生成逻辑

```javascript
class Star {
  constructor(x, y) {
    this.x = x;
    this.y = y;
    this.radius = 14;
    this.collected = false;
    this.bobPhase = Math.random() * Math.PI * 2; // 上下浮动相位
    this.glowPhase = Math.random() * Math.PI * 2; // 发光呼吸相位
  }

  update(dt) {
    this.bobPhase += 0.05;
    this.glowPhase += 0.08;
  }

  getDisplayY() {
    return this.y + Math.sin(this.bobPhase) * 4; // 上下浮动 4px
  }

  getGlowAlpha() {
    return 0.5 + Math.sin(this.glowPhase) * 0.3; // 0.2~0.8 发光呼吸
  }

  overlaps(px, py, pr) {
    const dx = px - this.x;
    const dy = py - this.getDisplayY();
    return Math.sqrt(dx*dx + dy*dy) < pr + this.radius;
  }
}
```

```javascript
// 生成时机：随平台生成时一起概率生成

// 收集检测（Game.js 每帧调用）

  }
}
```

```javascript
  ctx.fillStyle = '#FFFFFF';
  ctx.beginPath();
  ctx.arc(this.x, sy, 3, 0, Math.PI * 2);
```

## 7. 粒子特效

### 7.1 粒子类型

| 类型 | 触发时机 | 颜色 | 数量 | 行为 |
|------|---------|------|------|------|
| `bounce` | 碰平台 | 金/白色 | 8~12 | 向四周爆发，带重力 |
| `collect_star` | 收集星星 | 金色 | 15 | 以星星为中心向外爆发 |
| `trail` | 角色运动中 | 角色色 | 每3帧1个 | 无重力，快速消亡 |
| `death` | 游戏结束时 | 红色+白色 | 25 | 角色位置爆发 |

### 7.2 粒子复用

**可以直接复用** `Particle.js` 中的 `ParticleSystem`、`Particle`、`ScorePopup`。
只需要扩展 `emit()` 方法添加新类型：

```javascript
// ParticleSystem.emit() 新增 case
case 'bounce':
  return this._emitBounce(x, y, color || '#FFD700');
case 'collect_star':
  return this._emitCollectStar(x, y);
case 'trail':
  return this._emitTrail(x, y, color);

_emitBounce(x, y, color) {
  for (let i = 0; i < 10; i++) {
    const angle = -Math.PI / 2 + rand(-1.2, 1.2); // 主要向上
    const speed = rand(2, 5);
    this.particles.push(new Particle(
      x, y, color,
      Math.cos(angle) * speed,
      Math.sin(angle) * speed,
      rand(3, 7),
      randInt(15, 30),
      'circle', 0.18
    ));
  }
}

_emitCollectStar(x, y) {
  for (let i = 0; i < 15; i++) {
    const angle = rand(0, Math.PI * 2);
    const speed = rand(3, 8);
    this.particles.push(new Particle(
      x, y, '#FFD700',
      Math.cos(angle) * speed,
      Math.sin(angle) * speed,
      rand(4, 9),
      randInt(25, 45),
      'star', 0.05
    ));
  }
}
```

## 8. 视觉风格实现

### 8.1 背景（完全重写 Background.js）

```javascript
// 深蓝渐变背景（静态，不随相机滚动）
draw(ctx, camY) {
  // 背景永远从 y=0 开始，不受 camY 影响
  const grad = ctx.createLinearGradient(0, 0, 0, this.H);
  grad.addColorStop(0, '#0a1628');    // 顶部：深夜蓝
  grad.addColorStop(0.5, '#0f2040'); // 中间
  grad.addColorStop(1, '#1a2a4a');   // 底部：稍浅蓝
  ctx.fillStyle = grad;
  ctx.fillRect(0, 0, this.W, this.H);

  // 星空粒子（固定在背景，随时间缓慢漂移）
  this._drawStars(ctx);

  // 远处的紫色星云（半透明薄雾层）
  this._drawNebula(ctx);
}

_drawStars(ctx) {
  // 约 80 颗静态星星，部分闪烁
  for (const star of this.starField) {
    const flicker = star.flicker
      ? 0.4 + Math.sin(Date.now() * 0.003 + star.phase) * 0.4
      : 1.0;
    ctx.globalAlpha = flicker * star.brightness;
    ctx.fillStyle = star.color; // 白色或淡蓝/淡紫
    ctx.beginPath();
    ctx.arc(star.x, star.y, star.size, 0, Math.PI * 2);
    ctx.fill();
  }
  ctx.globalAlpha = 1;
}
```

### 8.2 角色视觉（完全重写 Player.js）

```javascript
// 荣耀糖糖角色：可爱发光糖果战士
// 形状：圆形糖果身体 + 小翅膀 + 闪亮眼睛
// 颜色：紫色主色 + 金色高光 + 粉色腮红

draw(ctx, camY) {
  const drawY = this.y + this.bobOffset - camY;

  ctx.save();
  ctx.translate(this.x, drawY);

  // 1. 金色外发光（始终存在）
  ctx.shadowColor = '#FFD700';
  ctx.shadowBlur = 25;

  // 2. 紫色翅膀（左右各一，小而精致）
  this._drawWings(ctx);

  // 3. 糖果身体（渐变紫色球体）
  const bodyGrad = ctx.createRadialGradient(-4, -5, 2, 0, 0, this.radius);
  bodyGrad.addColorStop(0, '#C44AFF');   // 高光：亮紫
  bodyGrad.addColorStop(0.5, '#8B2BE2'); // 中间：品紫
  bodyGrad.addColorStop(1, '#4B0082');   // 边缘：深紫
  ctx.beginPath();
  ctx.arc(0, 0, this.radius, 0, Math.PI * 2);
  ctx.fillStyle = bodyGrad;
  ctx.fill();

  // 4. 金色光环
  ctx.strokeStyle = 'rgba(255, 215, 0, 0.6)';
  ctx.lineWidth = 2;
  ctx.shadowBlur = 15;
  ctx.beginPath();
  ctx.arc(0, 0, this.radius + 4, 0, Math.PI * 2);
  ctx.stroke();

  // 5. 眼睛（可爱大眼）
  this._drawEyes(ctx);

  // 6. 腮红
  ctx.globalAlpha = 0.4;
  ctx.fillStyle = '#FF85C1';
  ctx.beginPath();
  ctx.ellipse(-10, 4, 5, 3, 0, 0, Math.PI * 2);
  ctx.fill();
  ctx.beginPath();
  ctx.ellipse(10, 4, 5, 3, 0, 0, Math.PI * 2);
  ctx.fill();

  ctx.beginPath();
  ctx.arc(0, 5, 3.5, 0, Math.PI);
  ctx.strokeStyle = '#2D0040';
  ctx.lineWidth = 1.5;
  ctx.stroke();

  ctx.restore();
}
```

## 11. 关键算法伪代码汇总

### 11.1 主循环

```javascript
_loop(timestamp) {
  const dt = Math.min((timestamp - this.lastTime) / 1000, 0.05);
  this.lastTime = timestamp;

  this._update(dt);   // 物理、碰撞、生成、相机
  this._draw();       // 渲染

  requestAnimationFrame(t => this._loop(t));
}
```

### 11.2 完整 Update

```javascript
_update(dt) {
  if (this.state === STATE.GAME_OVER) {
    this._deathTimer++;
    this.ps.update();
    this.shake.update();
    return;
  }

  // 1. 物理
  this.player.vy += GRAVITY * 60 * dt;
  this.player.vy = Math.min(this.player.vy, MAX_FALL_SPEED);
  this.player.vx *= FRICTION;
  this.player.x += this.player.vx;
  this.player.y += this.player.vy;

  // 2. 碰墙
  if (this.player.x < PLAYER_RADIUS) {
    this.player.x = PLAYER_RADIUS;
    this.player.vx = Math.abs(this.player.vx) * 0.7;
  }
  if (this.player.x > this.W - PLAYER_RADIUS) {
    this.player.x = this.W - PLAYER_RADIUS;
    this.player.vx = -Math.abs(this.player.vx) * 0.7;
  }

  // 3. 平台碰撞
  for (const plat of this.platforms) {
    plat.update(dt, this.W);
    if (plat.landsOn(this.player, this.player.vy)) {
      // 吸附 + 反弹
      this.player.y = plat.y - PLAYER_RADIUS;
      this.player.vy = -Math.abs(this.player.vy) * BOUNCE_DAMPING;
      this.ps.emit(this.player.x, this.player.y + PLAYER_RADIUS, 'bounce');
      playBounce();
    }
  }

  // 4. 星星碰撞
  for (const star of this.stars) {
    star.update(dt);
    if (!star.collected && star.overlaps(this.player.x, this.player.y, PLAYER_RADIUS)) {
      star.collected = true;
      this.score += 10;
      this.ps.emit(star.x, star.y, 'collect_star');
      this.ps.pop(star.x, star.y, '+10');
      playCollect();
    }
  }

  // 5. 相机跟随（只向上）
  const targetScreenY = this.H * PLAYER_SCREEN_RATIO;
  const targetCamY = this.player.y - targetScreenY;
  if (targetCamY < this.camY) {
    this.camY = lerp(this.camY, targetCamY, CAM_LERP);
  }

  // 6. 更新 highestY 并生成新平台
  if (this.player.y < this.highestY) this.highestY = this.player.y;
  if (this.nextPlatformY > this.highestY - this.H - 600) {
    this._generateMorePlatforms();
  }

  // 7. 清理
  this.platforms = this.platforms.filter(p => p.y < this.camY + this.H + 200);
  this.stars     = this.stars.filter(s => !s.collected && s.y < this.camY + this.H + 200);

  // 8. 游戏结束检测
  const playerScreenY = this.player.y - this.camY;
  if (playerScreenY > this.H + 80) {
    this._triggerGameOver();
  }

  this.ps.update();
  this.shake.update();
  this.hud.update(dt);
}
```

### 11.3 完整 Draw

```javascript
_draw() {
  const ctx = this.ctx;
  ctx.save();

  // 屏幕震动偏移
  const s = this.shake.getOffset();
  ctx.translate(s.x, s.y);

  // [1] 背景（屏幕空间，不受 camY 影响）
  this.bg.draw(ctx);

  // 应用相机变换（后续所有对象以 world 坐标绘制）
  ctx.translate(0, -this.camY);

  // [2] 平台层
  for (const plat of this.platforms) plat.draw(ctx, this.camY);

  // [3] 星星层
  for (const star of this.stars) star.draw(ctx, this.camY);

  // [4] 角色层
  if (this.player) this.player.draw(ctx, this.camY);

  // [5] 粒子层
  this.ps.draw(ctx, this.camY);

  ctx.restore();

  // [6] HUD 层（屏幕空间，ctx.restore 之后绘制）
  this.hud.draw(ctx);
}
```

## 12. 技术风险与注意事项

### 12.1 穿模问题
- 高速下落时可能一帧穿越 platform。用 `prevBottom <= platform.y + 2` 预判可缓解，但不能完全杜绝。
- 建议加 `MAX_FALL_SPEED = 18` 硬限制。

### 12.2 冷却防抖
- 触摸事件 `touchstart` 在移动端可能同时触发多次。100ms 冷却是必要的安全措施。

### 12.3 粒子性能
- 星空背景约 80 颗星星，全部直接绘制（不需要粒子系统）。
- 动态粒子（bounce/collect/trail）需要控制上限：bounce 和 collect 用完即删，trail 单次最多 20 个。

### 12.4 难度曲线
- 前期（Level 1-3）：平台宽（90~140px）、间距小（55~80px），玩家容易连跳建立信心。
- 中期（Level 4-7）：加入移动平台和更窄平台，增加挑战。
- 后期（Level 8+）：高间距（75~110px）、窄平台（45~80px），需要精准控制左右方向。
